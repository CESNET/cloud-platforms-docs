You can create your own unmanaged Kubernetes cluster with cluster-admin role.

Architecture:  
1x bastion server
1-3x master nodes
multiple workers

## Prerequisities
For kubernetes cluster you need openstack [group project](../technical-reference/brno-g2-site/get-access#group-project) with sufficient quotas and enabled flavors.

Example of quota requirements for simple testing cluster:
1x e1.small for bastion
1x r3.4core-8ram-30edisk for master nodes
1x c2.4core-16ram for worker nodes
1x floating IP 
250 GB of storage

Consider:
- How many control nodes do you need. For simple testing 1 is enough, for more serious bussiness you should have 3. 
- How many and how powerful worker nodes do you need - see [flavor types](https://docs.platforms.cloud.e-infra.cz/en/docs/technical-reference/brno-g2-site/flavors#flavor-types-and-naming-schema) and [flavor list](https://flavors-page-cloud-g2-10a73b324ea6f60888a8fd50d6fa6c9159aa60b27.gitlab-pages.ics.muni.cz/).
- If you need more public IPs for exposing application (e.g. nginx ingress) add 1 more floating IP. You need 1 floating IP for accesing the cluster.
- Add more storage if you need persistent volume

## Create new cluster

You can create cluster through [OpenOnDemand](https://ondemand.metacentrum.cz/pun/sys/dashboard/batch_connect/sys/bc_openstack_k8s/session_contexts/new).

## Accessing the cluster
### From localhost via command-line
After deployment is done you can download kubeconfig for accesing the cluster.
Install [kubectl](https://kubernetes.io/docs/tasks/tools/#kubectl) and store the kubeconfig to `~/.kube/config` and than, you can work with kubectl commands.

## For advanced users
If you need more complex solution than OpenOnDemand currently provides, you can create KOST via CLI following steps in this [repo](https://gitlab.ics.muni.cz/cloud/kubernetes/kubernetes-infra-example). Or you can contact our [support](../technical-reference/brno-g2-site/get-support).
